# BLE 通信协议

本文档记录 SmartSleep 设备与上位机（Flutter App）之间的 BLE 通信协议。

## 基础信息

- **BLE 服务 UUID**: `fff0`
- **写入特征 UUID**: `fff2`
- **通知特征 UUID**: `fff1`
- **MTU**: 247 字节
- **字节序**: 所有多字节数据均为小端序（Little-Endian，低位在前高位在后）

## 帧格式

### 发送帧（上位机 → 设备）

```
7D [类型] [长度] 00 55 4E 43 4F 4E 46 49 47 45 44 [负载] 0D
```

| 字节位置 | 值 | 说明 |
|---------|-----|------|
| 0 | 0x7D | 帧头固定标识 |
| 1 | 0xNN | 命令类型 |
| 2 | 0x0F/0x10 | 帧长度（无负载=0x0F=15，有负载=0x10=16） |
| 3 | 0x00 | 保留 |
| 4~13 | `55 4E 43 4F 4E 46 49 47 45 44` | 固定签名 "UNCONFIGED" |
| 14 | [负载] | 有负载时为数据字节 |
| 15 | 0x0D | 帧尾固定标识 |

### 响应帧（设备 → 上位机）

```
帧头 数据类型 [内容...]
```

- `bytes[0]` = 帧头标识
- `bytes[1]` = 数据类型（响应类型码）
- `bytes[2..]` = 响应内容（视类型而定）

> **例外**: 设备信息帧以 `A5 5A` 开头，固定 11 字节，由 `DeviceInfo.fromBytes()` 解析。

### 多包响应缓冲策略

BLE MTU=247，较大的响应（如 0x81/0x82 的 360~450 字节）会拆分为多个通知包。缓冲策略：

1. 收到首包时开始缓冲（跳过帧头+类型 2 字节，只缓存数据部分）
2. 累积到预期长度后立即解析
3. 若 500ms 内无新包到达，也尝试解析已有数据

## 命令列表

### 0x01 — 重传指令（120秒）

**用途**: 请求设备重传过去 120 秒的历史秒级数据

**发送**: 固定 15 字节
```
7D 01 0F 00 55 4E 43 4F 4E 46 49 47 45 44 0D
```

**响应**: 类型 `0x81`，30 组 × 12 字节 = 360 字节

**每组结构（12字节）**:

| 偏移 | 长度 | 字段 | 说明 |
|------|------|------|------|
| 0 | 1 | 序列号 | |
| 1 | 4 | 时间戳 | Unix 时间戳，小端序。0x00000000 = 数据无效 |
| 5 | 1 | 状态 | |
| 6 | 1 | 心率 | |
| 7 | 1 | 呼吸率 | |
| 8 | 2 | SDATA | 体动数据 |
| 10 | 2 | PDATA | 压力数据 |

**模型**: `RetransmitRecord`
**测试卡片**: `RetransmitTestCard`

---

### 0x02 — 重传指令（30分钟）

**用途**: 请求设备重传过去 30 分钟的历史分钟级数据

**发送**: 固定 15 字节
```
7D 02 0F 00 55 4E 43 4F 4E 46 49 47 45 44 0D
```

**响应**: 类型 `0x82`，30 组 × 15 字节 = 450 字节

**每组结构（15字节）**:

| 偏移 | 长度 | 字段 | 说明 |
|------|------|------|------|
| 0 | 1 | 序列号 | |
| 1 | 4 | 时间戳 | Unix 时间戳，小端序 |
| 5 | 1 | 状态 | |
| 6 | 1 | 心率 | |
| 7 | 1 | 呼吸率 | |
| 8 | 1 | 体动 | |
| 9 | 1 | 打鼾次数 | |
| 10 | 1 | 呼吸障碍 | |
| 11 | 2 | PTHD | |
| 13 | 2 | 温度 | |

**模型**: `Retransmit30Record`
**测试卡片**: `Retransmit30TestCard`

---

### 0x03 — 停止指令

**用途**: 强制停止设备工作，进入待机模式

**发送**: 固定 15 字节
```
7D 03 0F 00 55 4E 43 4F 4E 46 49 47 45 44 0D
```

**响应**: 类型 `0x83`，无内容

**测试卡片**: `StopCommandTestCard`

---

### 0x04 — 工作模式设定

**用途**: 设置设备工作模式

**发送**: 16 字节（14 字节前缀 + 模式字节 + 0x0D）
```
7D 04 10 00 55 4E 43 4F 4E 46 49 47 45 44 [模式] 0D
```

**模式字节**:

| 值 | 模式 | 说明 |
|----|------|------|
| 0x20 | 上位机监控模式 | 设备向 App 推送实时数据（0x85/0x86） |
| 0x30 | 数据调试模式 | 调试用 |

**响应**: 类型 `0x84`，1 字节确认码

| 确认码 | 含义 |
|--------|------|
| 0x00 | 执行成功 |
| 0x01 | 不支持该工作模式 |
| 0x02 | 未就绪 |
| 0x03 | 帧长度错误 |
| 0x04 | 参数错误 |

**测试卡片**: `ModeCommandTestCard`

---

### 0x05 — 实时秒数据流

**用途**: 设备在监控模式下每秒自动推送的实时数据

**发送**: 无需发送（由 0x04 设定监控模式后自动推送）

**响应**: 类型 `0x85`，12 字节/秒，结构同 `RetransmitRecord`

**存储**: BleDataState 中 `latestSecondRecord`（最新一条）+ `secondRecords`（环形缓冲，最多 120 条）

---

### 0x06 — 实时分钟数据流

**用途**: 设备在监控模式下每分钟自动推送的实时数据

**发送**: 无需发送（由 0x04 设定监控模式后自动推送）

**响应**: 类型 `0x86`，15 字节/分钟，结构同 `Retransmit30Record`

**存储**: BleDataState 中 `latestMinuteRecord`（最新一条）+ `minuteRecords`（环形缓冲，最多 30 条）

---

### 0x07 — 设备状态查询

**用途**: 查询设备当前工作状态

**发送**: 固定 15 字节
```
7D 07 0F 00 55 4E 43 4F 4E 46 49 47 45 44 0D
```

**响应**: 类型 `0x87`，6 字节

| 偏移 | 长度 | 字段 | 说明 |
|------|------|------|------|
| 0 | 1 | 工作模式 | 0x00=待机, 0x20=监控, 0x30=调试, 0x31=BLE调试, 0xF0=固件升级 |
| 1 | 1 | 错误码 | 0x00=正常 |
| 2 | 4 | 时间戳 | 设备校准时间，Unix 时间戳 LE |

**模型**: `DeviceStatus`（含 `modeName` 中文模式名、`hasError` 等便利属性）
**测试卡片**: `DeviceStatusTestCard`

---

### 0x08 — 心跳应答

**用途**: 保持与设备的连接活跃。建议每分钟发送一次，超 5 分钟未收到设备将重启 WiFi

**发送**: 固定 15 字节
```
7D 08 0F 00 55 4E 43 4F 4E 46 49 47 45 44 0D
```

**响应**: 类型 `0x88`，无内容

**测试卡片**: `HeartbeatTestCard`（支持手动发送 + 自动1分钟定时发送）

---

### 0x09 — 压力校准指令

**用途**: 触发设备压力传感器校准。设备强制停止工作并开始校准，约 4 秒后返回结果

**发送**: 固定 15 字节
```
7D 09 0F 00 55 4E 43 4F 4E 46 49 47 45 44 0D
```

**响应**: 类型 `0x89`，1 字节结果

| 结果 | 含义 |
|------|------|
| 0x00 | 校准完成 |

**测试卡片**: `PressureCalibrateTestCard`

---

### 0x0A — 参数管理指令

**用途**: 读取、复位或设置设备的 16 个工作参数

**发送**: 16 字节（14 字节前缀 + 子命令 + 0x0D）

**子命令**:

| 子命令 | 帧格式 | 说明 |
|--------|--------|------|
| 0x01 复位 | `... 44 01 0D` | 恢复默认参数 |
| 0x02 读取 | `... 44 02 0D` | 读取当前参数 |
| 0x03 设置 | `... 44 [64字节参数] 0D` | 设置参数（帧长 80 字节） |

**响应**: 类型 `0x8A`

- 复位（0x01）/ 设置（0x03）：无额外内容，`bytes[16]` = 子命令
- 读取（0x02）：`bytes[2..65]` = 64 字节参数数据，`bytes[16]` = 子命令

**参数结构**: 16 个 float（32位浮点数，小端序），共 64 字节

| 序号 | 名称 | 默认值 | 说明 |
|------|------|--------|------|
| P1 | 离入床动态阈值 | 自动调整 | |
| P2 | 体动动态阈值 | 自动调整 | |
| P3 | 离入床静态阈值 | 自动调整 | |
| P4 | 打鼾灵敏度 | 18.5 | |
| P5 | 弱呼吸灵敏度 | 5.0 | |
| P6 | 实时分期阈值① | 2.3 | |
| P7 | 实时分期阈值② | 17.0 | |
| P8 | 实时分期阈值③ | 4.0 | |
| P9 | 整晚分期阈值① | 13.0 | |
| P10 | 整晚分期阈值② | 3.0 | |
| P11 | 分期算法阈值 | 9.3 | |
| P12 | 报表生成时间 | 9.0 | 0~23 小时 |
| P13 | 自动调整 | 自动调整 | |
| P14 | 分期算法阈值 | 5.0 | |
| P15 | 备用参数 | 1.0 | |
| P16 | 硬件参数 | 1.0 | 勿改 |

**模型**: `DeviceParameters`（`fromBytes()` / `toBytes()` 序列化）
**测试卡片**: `ParameterTestCard`

---

### 0x0B — 校准时钟指令

**用途**: 将上位机的系统时间同步到设备

**发送**: 16 字节（14 字节前缀 + 4 字节 Unix 时间戳 LE + 0x0D）
```
7D 0B 10 00 55 4E 43 4F 4E 46 49 47 45 44 [时间戳4B LE] 0D
```

时间戳为标准 Unix 时间戳（秒），小端序编码。

**响应**: 类型 `0x8B`，无内容

**测试卡片**: `ClockCalibrateTestCard`

---

### 0x0C — 固件版本查询

**用途**: 查询设备固件版本字符串

**发送**: 固定 15 字节
```
7D 0C 0F 00 55 4E 43 4F 4E 46 49 47 45 44 0D
```

**响应**: 类型 `0x8C`，`bytes[2..]` 为 UTF-8 版本字符串

示例: `UiSleep_Pro_BLE_HVER810_FVER180`

**存储**: BleDataState.firmwareVersion，连接成功后自动查询

---

### 0x17 — 恢复出厂设置

**用途**: 恢复设备出厂默认设置

**发送**: 固定 1 字节
```
17
```

**响应**: 类型 `0x97`，无内容

**集成**: `FactoryResetCubit` → `BleDataCubit.sendCommand()`
**UI**: profile_page 中的恢复出厂设置按钮

---

### A5 5A — 设备信息（自动推送）

**用途**: 设备连接后自动推送的设备信息帧

**帧结构**: 固定 11 字节，以 `A5 5A` 开头

**模型**: `DeviceInfo`（含温度、模式、状态等字段）
**存储**: BleDataState.deviceInfo
